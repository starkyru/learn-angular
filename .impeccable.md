## Design Context

### Users

Developers preparing for Angular interviews who need a runnable, focused reference while
studying concepts and completing small exercises.

### Brand Personality

Calm, precise, encouraging. The interface should build confidence without competing with
the lessons themselves.

### Aesthetic Direction

Light editorial notebook: warm paper-like surfaces, ink-toned text, and a restrained teal
accent. Use readable system typography, clear hierarchy, and no decorative gradients or
card-heavy dashboard treatment.

### Design Principles

1. Keep the learning path obvious and each lesson one click away.
2. Prioritize readable code-adjacent content, contrast, and keyboard navigation.
3. Use color only to orient and communicate state; never as the sole signal.
4. Prefer a small, responsive layout system over decorative UI.
5. Respect reduced-motion preferences and keep interaction feedback purposeful.
